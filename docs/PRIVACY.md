# 隐私说明

DocsFabrica 是一款运行在 Microsoft Word 中的加载项，会根据用户指令处理文档内容。

## 数据访问

- 用户使用加载项时，DocsFabrica 会读取当前打开的 Word 文档。
- DocsFabrica 可能会读取用户明确选择为参考文件的本地文件。
- DocsFabrica 不会主动读取用户未选择的任意文件。

## AI 服务请求

当用户发送命令时，相关文档内容、已选择的参考文件内容以及用户命令，可能会被发送至用户配置的 AI 模型服务商。

当前已完成适配与测试的模型服务商为 DeepSeek。其他模型服务商入口为后续兼容性预留，当前暂不建议使用。

## API Key

API Key 仅用于调用用户选择的模型服务商。用户不应在截图、GitHub Issues、支持请求或公开文档中分享 API Key。

## 本地文件

DocsFabrica 可能会创建本地备份、本地日志以及安装和运行所需的本地服务文件。

## 用户责任

用户应自行确认使用 DocsFabrica 是否符合其所在组织、机构或司法辖区的数据安全和隐私要求。

除非所选 AI 服务商及使用场景已经获得相关组织批准，否则请勿使用 DocsFabrica 处理机密、受监管或敏感文档。

# Privacy Policy

DocsFabrica is a Microsoft Word add-in that processes document content according to user commands.

## Data Access

- DocsFabrica reads the currently opened Word document when the add-in is used.
- DocsFabrica may read files explicitly selected by the user as reference files.
- DocsFabrica does not intentionally read arbitrary files that the user has not selected.

## AI Service Requests

When the user sends a command, relevant document content, selected reference content, and the user command may be sent to the configured AI model service provider.

The current tested model provider is DeepSeek. Other provider entries are reserved for future compatibility and are not recommended for current use.

## API Keys

API Keys are used only for calling the selected model service provider. Users should not share API Keys in screenshots, GitHub Issues, support requests, or public documents.

## Local Files

DocsFabrica may create local backups, local logs, and local service files required for installation and operation.

## User Responsibility

Users are responsible for ensuring that their use of DocsFabrica complies with their organization, institution, or jurisdiction's data security and privacy requirements.

Do not use DocsFabrica to process confidential, regulated, or sensitive documents unless the selected AI service provider and use case have been approved by the relevant organization.
